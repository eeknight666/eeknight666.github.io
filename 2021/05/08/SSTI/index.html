SSTI

漏洞简介

SSTI就是服务器端模板注入(Server-Side Template Injection)

SSTI是先获取了一个输入，再后端的渲染处理上进行了语句的拼接，然后执行。SSTI利用的是现在的网站模板引擎主要是Python的jinja2 mako tornado django，php的smarty twig，java的jade velocity。当这些框架对运用渲染函数生成html的时候会出现SSTI的问题。

前置知识

模板引擎

模板引擎（这里特指用于Web开发的模板引擎）是为了使用户界面与业务数据（内容）分离而产生的，它可以生成特定格式的文档，用于网站的模板引擎就会生成一个标准的HTML文档。
模板引擎可以让（网站）程序实现界面与数据分离，业务代码与逻辑代码的分离，这就大大提升了开发效率，良好的设计也使得代码重用变得更加容易。

也就是说，利用模板引擎来生成前端的html代码，模板引擎会提供一套生成html代码的程序，然后只需要获取用户的数据，然后放到渲染函数里，然后生成模板+用户数据的前端html页面，然后反馈给浏览器，呈现在用户面前。

魔术方法

在正式学习SSTI前还要了解一些python的类的知识。

  • __class__

    用于返回对象所属的类

    Str(字符串)、dict(字典)、tuple(元组)、list(列表)

  • __base__

    以字符串的形式返回一个类所继承的类

  • __bases__

    用来查看类的基类，也可是使用数组索引来查看特定位置的值

  • __mro__

    和上面两个一样，可以用来获取基类

    注意，后面是要跟-1

  • __subclasses__()

    获取当前类的所有子类

    在和上面的类组合用时，可以用[]来指定某个类

  • __init__

    所有自带带类都包含init方法，常用他当跳板来调用globals

  • __globals__

    会以字典类型返回当前位置的全部模块，方法和全局变量，用于配合init使用

    在返回的字典中找到对应的值，再去值就可以了

基本模板

**{ % ... % }**

{ % % } 主要用来声明变量，也可以用于条件语句和循环语句。

**{ { ... } }**

{ { } }来表示变量名，这种{ { } }语法叫做变量代码块。

可以将表达式打印到模板输出，比如我们一般在里面输入2-1，2*2，或者是字符串，调用对象的方法，都会渲染出结果

也可以显示出字典或者列表的某个元素

{ # ... # }

漏洞成因在于：**render_template_string函数在渲染模板的时候使用了%s来动态的替换字符串，我们知道Flask 中使用了Jinja2 作为模板渲染引擎，{{}}`在Jinja2中作为变量包裹标识符，Jinja2在渲染的时候会把`{{}}包裹的内容当做变量解析替换。比如{{1+1}}会被解析成2。**

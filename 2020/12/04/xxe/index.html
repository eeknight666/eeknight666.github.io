XXE漏洞

XXE简介

XXE漏洞全称XML External Entity Injection即xml外部实体注入漏洞，XXE漏洞发生在应用程序解析XML输入时，没有禁止外部实体的加载，导致可加载恶意外部文件，造成文件读取、命令执行、内网端口扫描、攻击内网网站、发起dos攻击等危害。

或者说：XXE主要是利用了DTD引用外部实体导致的漏洞，所以学习XXE漏洞的重点就是DTD外部实体。

XML组成结构

学习XXE漏洞的重点就是XML

XML用于标记电子文件使其具有结构性的标记语言，可以用来标记数据、定义数据类型，是一种允许用户对自己的标记语言进行定义的源语言。

XML文档结构包括XML声明、DTD文档类型定义（可选）、文档元素

PCDATA

PCDATA 的意思是被解析的字符数据（parsed character data）。
PCDATA 是会被解析器解析的文本。这些文本将被解析器检查实体以及标记。

CDATA

CDATA 的意思是字符数据（character data）。
CDATA 是不会被解析器解析的文本。

文档类型定义(DTD)

DTD是用来控制XML的格式规范的。

DTD实体是用于定义引用普通文本或特殊字符的快捷方式的变量，可以在XML内部声明，也可以外部引用。

一些关键字解释：

  • DOCTYPE（DTD的声明）
  • ENTITY （实体的声明）
  • SYSTEM、PUBLIC（外部资源申请）

其他

CDATA绕过

如果要读取的文件内容有<> &，被解析器解析出错了怎么办？

这时候可以用CDATA来绕过。

他的基本原理：CDATA 是不会被解析器解析的文本。

有些内容可能不想让解析引擎解析执行，而是当做原始的内容处理，用于把整段数据解析为纯字符数据而不是标记的情况包含大量的 <> & 或者
“ 字符，CDATA节中的所有字符都会被当做元素字符数据的常量部分，而不是 xml标记

特别提及

XML特征

  • 所有 XML 元素都须有关闭标签。
  • XML 标签对大小写敏感。
  • XML 文档必须有根元素。
  • XML 的属性值须加引号。
  • 在 XML 中，空格会被保留,多个空格不会被合并为一个。
